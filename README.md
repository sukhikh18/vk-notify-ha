# VK Notify for Home Assistant

Интеграция `VK Notify` это пользовательская интеграция, которая позволяет отправлять уведомления из Home Assistant в сообщения ВКонтакте (в личные сообщения от имени сообщества, чаты сообщества)
<!-- и принимать входящие callback-события VK через webhook. -->

Добавляет:

- сервис `vk_notify.send_message` — отправка текста;
- сервис `vk_notify.send_photo` — отправка фото;
- сервис `vk_notify.send_document` — отправка файла/видео как документ;
- сервис `vk_notify.send_video` — отправка нативного video-вложения (требует user token);
<!-- - событие `vk_notify_received` при входящих webhook-событиях VK Callback API. (beta*) -->

## Как установить в HA

### Вариант А: Через HACS (рекомендуется)

1. Откройте HACS → `Интеграции`. (/hacs/dashboard)
2. Нажмите `⋮` → `Пользовательские репозитории`.
3. Добавьте URL репозитория и тип `Интеграция`.
4. Установите `VK Notify` кнопкой `Добавить`.
5. Перезапустите Home Assistant.

### Вариант Б: Вручную

1. Скопируйте папку `custom_components/vk_notify` в:
   - `/config/custom_components/vk_notify`
2. Перезапустите Home Assistant.

## Как настроить ВК

### Получить токен сообщества

1. Откройте/создайте сообщество VK. (рекомендую в настройках указать Тип сообщества: Частное)
2. `Управление` → `Дополнительно` → `Работа с API` → `Создать ключ`.
3. Создайте ключ доступа сообщества.
Необходимые права:
   - Доступ к сообщениям
   - Доступ к фотографиям (если планируется отправлять фото с помощью `send_photo`)
   - Доступ к документам (если планируется отправлять документы с помощью `send_document`)

### Узнать ИД сообщества

Получить ID сообщества можно из URL веб версии (к примеру `https://vk.com/club234567891?act=tokens` → `ID сообщества = 234567891`).
Число после Club является идентификатором вашего сообщества

### Узнать ИД пользователя

Так же можно из URL профиля или URL аватара, где первое число после photo является идентификатором (к примеру: `https://vk.com/user?z=photo123456789_155261320%2Falbum123412345_0%2Frev` → `ID пользователя = 123456789`)

## Как настроить HA

### Создать интеграцию

`Настройки` → `Устройства и службы` → `Добавить интеграцию` → `VK Notify`.

В форме заполните:

- `Токен доступа` — Токен доступа (сообщества);
- `group_id` — ID сообщества;
- `recipient_id` — получатель по умолчанию (`peer_id`);
- `receive_mode`:
  - `send_only` — только отправка,
<!-- 
  - `webhook` — отправка + прием callback; (beta*)
- `webhook_secret` и `confirmation_code` — если включен webhook.
-->

### Что такое `recipient_id` (peer_id)

- Если не указать: Используется по умолчанию (что был указан при добавлении интеграции)
- Можно указать чат сообщества: К примеру `2000000001` - где 1 порядковый номер чата сообщества.

Пример:
- беседа № 15 → `recipient_id: 2000000015`

> Для `vk_notify.send_video` нужен video_access_token (**token пользователя**, не сообщества) с правами `video,messages`.
Потому что VK не разрешает сохранять видео сообществу

_Если вы не можете получить пользовательский токен, используйте `vk_notify.send_document`._

### Примеры сервисов

#### Текст

```yaml
service: vk_notify.send_message
data:
  message: "Сигнализация включена"
  recipient_id: 2000000015
```

#### Фото

```yaml
service: vk_notify.send_photo
data:
  file: "/config/www/camera.jpg"
  caption: "👤 Обнаружен человек"
  recipient_id: 2000000015
```

#### Документ (в том числе видео как файл)

```yaml
service: vk_notify.send_document
data:
  file: "/config/www/clip.mp4"
  caption: "Запись с камеры"
  recipient_id: 2000000015
```

#### Нативное видео (playable в VK)

```yaml
service: vk_notify.send_video
data:
  file: "/config/www/clip.mp4"
  caption: "🚗 Обнаружен транспорт"
  recipient_id: 2000000015
  video_access_token: "vk1.a...."
```
